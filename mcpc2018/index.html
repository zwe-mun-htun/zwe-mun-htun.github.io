2018 Myanmar Collegiate
Programming Contest
(MCPC 2018)

Hosted by University of Computer Studies, Yangon, Myanmar


UCSY will host the third 2018 ACM-ICPC Asia-Yangon Regional Programming Contest under auspices of ACM (Association for Computing Machinery). Prior to Asia level contest, UCSY will host national level contest, 2018 Myanmar Collegiate Programming Contest on August 11, 2018 (Saturday). The winner teams of Myanmar Collegiate Prgramming Contestare eligible to participate in 2018 Asia-Yangon Regional Programming Contest hosted by UCSY.

Registration and Contest Dates

Registration Opening: September 25, 2017 (Monday)
Registration Closing: November 15, 2017 (Wednesday)
Contest Rehearsal: December 9, 2017 (Saturday)
Contest Day: December 10, 2017 (Sunday)

Team Registration: Asia-Yangon Regional Page
The Scoreboard : Scoreboard Page
Previous Contest: Asia-Yangon-2016

Registration Fee

Foreign Team Registration Fee: US$ 200

  • For a team with 3 contestants and 1 coach

  • Extra US$ 35 for each extra person

Local Team Registration Fee: 100,000 Kyats

  • For a team with 3 contestants and 1 coach

  • Extra 30,000 Kyats for each extra person

Only on-site registration payment will be accepted.

Ubuntu 16.04 LTS

Editors
Geany, gedit, Sublime Text

IDEs for JAVA
Eclipse, NetBeans

Languages
Java (jdk 1.7.0 and above)

C/C++
GNU C++ compiler : g++ 4:5.3.1-ubuntu1
GNU C compiler : gcc 4:5.3.1-ubuntu1

Judging System
PC2


ၿပိဳင္ပြဲ၀င္မ်ားႏွင့္ အသင္းမ်ား အရည္အခ်င္းသတ္မွတ္ခ်က္

  • ၿပိဳင္ပြဲ၀င္(contestants) မ်ားသည္ မိမိတို႔ကိုယ္စားျပဳ ယွဥ္ၿပိဳင္မည့္ တကၠသိုလ္၊ ေကာလိပ္တြင္ တတ္ေရာက္ သင္ၾကားေနသူ ေက်ာင္းသား၊ ေက်ာင္းသူမ်ား ျဖစ္ရပါမည္။

  • ၿပိဳင္ပြဲ၀င္မ်ားသည္ (၁၉၉၄)ခုႏွစ္ႏွင့္ ေနာက္ပိုင္းတြင္ ေမြးဖြားသူ ျဖစ္ရပါမည္။

  • ၿပိဳင္ပြဲတြင္ အသင္းလိုက္ ယွဥ္ၿပိဳင္ရမည္ျဖစ္ၿပီး အသင္းတစ္သင္းလွ်င္ တကၠသိုလ္၊ေကာလိပ္ တစ္ခုတည္းတြင္ တတ္ေရာက္ေနေသာ ၿပိဳင္ပြဲ၀င္ (၃)ဦး ပါ၀င္ရပါမည္။ အသင္းကို မိမိတို႔ႏွစ္သက္ရာ အမည္ေပးႏိုင္ပါသည္။
    (ဥပမာ UCSY-Digit, bYtEs, 1100011, etc)

  • တကၠသိုလ္၊ေကာလိပ္ တစ္ခုစီသည္ မိမိတို႔အတြက္ သတ္မွတ္ထားေသာ အသင္းအေရအတြက္ထက္ မပိုေစဘဲ ၿပိဳင္ပြဲ၀င္အသင္းမ်ား ေစလႊတ္ႏိုင္ပါသည္။

  • ၿပိဳင္ပြဲ၀င္တစ္သင္းလွ်င္ အသင္းႀကီးၾကပ္သူ(coach) အေနျဖင့္ တကၠသိုလ္၊ေကာလိပ္ သင္ၾကားေရး ၀န္ထမ္းတစ္ဦး လုိက္ပါရန္ လိုအပ္ပါသည္။ အသင္းတစ္သင္းထက္ပိုေသာ တကၠသိုလ္၊ေကာလိပ္မ်ား အေနျဖင့္ အနည္းဆံုး ႀကီးၾကပ္သူတစ္ဦး လိုက္ပါရပါမည္။


    • ၿပိဳင္ပြဲ၀င္အသင္းမ်ားကို အဆင့္သတ္မွတ္ျခင္း

  • အဆင့္သတ္မွတ္ရာတြင္ မိမိအသင္းမွ ေျဖရွင္းႏိုင္ေသာ Problem အေရအတြက္ အမ်ားအနည္းအလိုက္ သတ္မွတ္ပါမည္။

  • ေျဖရွင္းႏိုင္ေသာ Problem အေရအတြက္ တူေနပါက Problem ေျဖရွင္းရန္ၾကာေသာ အခ်ိန္စုစုေပါင္း ျဖင့္ အဆင့္ သတ္မွတ္ပါမည္။

  • အထက္ပါ (၂) ခ်က္ႏွင့္ တူညီေနပါက ေနာက္ဆံုးေျဖရွင္းခဲ့ေသာ Problem ကို ေျဖရွင္းသည့္ အခ်ိန္ျဖင့္ ဆံုးျဖတ္ပါမည္။

  • အကယ္၍ ထပ္တူေနေသးပါက ဒုတိယေနာက္ဆံုး Problem ေျဖရွင္းသည့္ အခ်ိန္၊ တတိယေနာက္ဆံုး Problem ေျဖရွင္းသည့္ အခ်ိန္၊ စတုတၳေနာက္ဆံုး Problem ေျဖရွင္းသည့္ အခ်ိန္ စသည္ျဖင့္ အဆံုး အျဖတ္ေပးသြားမည္ ျဖစ္ပါသည္။


    အေထြေထြအေၾကာင္းအရာမ်ား

  • ၿပိဳင္ပြဲႏွင့္ပတ္သက္၍ အဆင္မေျပမႈႏွင့္ မေက်နပ္ခ်က္မ်ားရွိပါက ၿပိဳင္ပြဲက်င္းပေရးအဖြဲ႔ႏွင့္ Judge ဒိုင္မ်ားထံတြင္ ၿပိဳင္ပြဲၿပီးခ်ိန္မွစ၍ မိနစ္ ၃၀ အတြင္း တင္ျပႏိုင္ပါသည္။

  • ၿပိဳင္ပြဲအတြင္း မေမွ်ာ္လင့္ေသာ အေျခအေနမ်ားေၾကာင့္ ေႏွာင့္ေႏွးမႈမ်ားရွိပါက ၿပိဳင္ပြဲက်င္းပေရးအဖြဲ႔ႏွင့္ Judge ဒိုင္မ်ားအဖြဲ႕မွ ဆံုးျဖတ္ေျဖရွင္းေပးပါမည္။

  • ၿပိဳင္ပြဲႏွင့္ပတ္သက္သည့္ ေနာက္ဆံုးအခ်က္အလက္မ်ား၊ သတင္းမ်ားႏွင့္ ဓာတ္ပံုမ်ားကို MCPC 2017 ၏ တရား၀င္ Facebook Page တြင္ ၀င္ေရာက္ၾကည့္႐ႈႏိုင္ပါသည္။

  • MCPC 2017 ႏွင့္ပတ္သက္ေသာ ကိစၥရပ္မ်ားအတြက္ တရား၀င္Email (icpc@ucsy.edu.mm) ကိုသာ ဆက္သြယ္ေဆာင္ရြက္ ေပးေစလိုပါသည္။


    • ၿပိဳင္ပြဲမတိုင္ခင္

  • ၿပိဳင္ပြဲက်င္းပရာခန္းမအတြင္းသို႔ Electronic Devices (Calculator, Mobile Phone, USB stick, etc) မ်ားကို ယူေဆာင္ခြင့္ မျပဳပါ။

  • အသင္းတစ္သင္းလွ်င္ မိမိတို႔ႏွစ္သက္ရာ Dictionary စာအုပ္တစ္အုပ္ႏွင့္ အသံုးျပဳရန္ Stationary (pen, pencil, ruler, etc) မ်ားကို ယူေဆာင္ ခြင့္ျပဳပါသည္။ ယင္းတို႔ကို ပထမရက္ အစမ္း ေလ့က်င့္ခ်ိန္ တြင္ ယူေဆာင္လာခဲ့ ရမည္ျဖစ္ျပီး ၿပိဳင္ပြဲ က်င္းပရာ ခန္းမအတြင္း ထားခဲ့ရမည္ ျဖစ္ပါသည္။

  • အၾကမ္းအသံုးျပဳရန္အတြက္ အသင္းတစ္သင္းကို A4 စာရြက္ (၁၅) ရြက္ကို ၿပိဳင္ပြဲက်င္းပသည့္ေန႔တြင္ ထုတ္ေပးထားပါမည္။

  • ထို႔ျပင္ A4 (သို႔) Legel စာမ်က္ႏွာ (၂၅)မ်က္ႏွာထိ Team Reference Document အျဖစ္ယူခြင့္ျပဳပါသည္။ ယင္းစာမ်က္ႏွာမ်ားေပၚတြင္ မိမိတို႔ကိုယ္ပိုင္မွတ္စုမ်ား၊ ပံုႏွိပ္စာမ်ား ယူလာႏိုင္ၿပီး အစမ္းေလ့က်င့္ၿပီးခ်ိန္တြင္ ခန္းမတြင္း ထားခဲ့ရပါမည္။

  • အသင္းအားလံုးသည္ တူညီေသာ Hardware/Software တို႔ျဖင့္ ယွဥ္ၿပိဳင္ၾကရမည္။ မည္သည့္ အပို Hardware/Software မွ တပ္ဆင္အသံုးျပဳခြင့္ မျပဳပါ။

  • ၿပိဳင္ပြဲ၀င္အသင္းမ်ားသည္ သတ္မွတ္ထားေသာ ၀တ္စံုကို၀တ္ဆင္၍ ၿပိဳင္ပြဲ၀င္ၾကရပါမည္။ ယင္း၀တ္စံုမ်ားကို ပထမေန႔ ၿပိဳင္ပြဲ၀င္အသင္းမ်ား မွတ္ပံုတင္ Registration ခ်ိန္တြင္ ထုတ္ေပးပါမည္။

  • ၿပိဳင္ပြဲက်င္းပရာခန္းမအတြင္းသို႔ Team member (၃)ဦးသာ ၀င္ခြင့္ ျပဳပါမည္။

  • အသင္းႀကီးၾကပ္သူ(coach)မ်ားသည္ အစမ္းယွဥ္ၿပိဳင္ခ်ိန္ ေနာက္ပိုင္း (၁)နာရီတြင္ ၿပိဳင္ပြဲခန္းမ အတြင္းသို႔ ၀င္ေရာက္ အားေပး ႏိုင္ေသာ္လည္း ၿပိဳင္ပြဲေန႔တြင္ ခန္းမအတြင္းသို႔ ၀င္ခြင့္မျပဳပါ။


    • ၿပိဳင္ပြဲက်င္းပစဥ္

  • ၿပိဳင္ပြဲတြင္ အနည္းဆံုး Problem (၁၀)ပုဒ္ကို (၅) နာရီအတြင္း ေျဖရွင္းရမည္ျဖစ္သည္။

  • ၿပိဳင္ပြဲတစ္ခုလံုးကို PC2 Auto Judging System အသံုးျပဳ၍ က်င္းပမည္ျဖစ္သည္။

  • Problem တစ္ခုေျဖရွင္းၿပီးတိုင္း ယင္း Source file ကို ခ်က္ျခင္း စစ္ေဆးရန္ Judge မ်ားထံသို႔ ေပးပို႔ ရမည္။ Judge ထံမွ မွန္/မမွန္ အေျဖကို အခ်ိန္အနည္းငယ္အတြင္း သိရွိႏိုင္ပါသည္။ မွန္ကန္ပါက ရရွိထားေသာ အဆင့္သည္ ခ်က္ျခင္း ေျပာင္းလဲသြားမည္ ျဖစ္ပါသည္။

  • မမွန္ကန္သည့္ အေျဖေပးပို႔မႈအတြက္ ျပစ္ဒါဏ္အျဖစ္ ေပးပို႔မႈတစ္ခုလွ်င္ မိနစ္(၂၀) ကို ယင္းပုစာၦမွန္သည့္ အခ်ိန္တြင္ ထည့္ေပါင္းခံရပါမည္။ Compilation Error အတြက္ မည္သည့္ ျပစ္ဒါဏ္မွမယူပါ။

  • ၿပိဳင္ပြဲ၀င္ေနစဥ္ အသင္းတစ္သင္းႏွင့္ တစ္သင္း စကားေျပာဆိုဆက္သြယ္ျခင္း လံုး၀မျပဳလုပ္ရပါ။

  • ၿပိဳင္ပြဲက်င္းပခ်ိန္အတြင္း Print ထုတ္လိုပါက အသင္းတစ္သင္းလွ်င္ အမ်ားဆံုး A4 ၁၅မ်က္ႏွာအထိ ၿပိဳင္ပြဲခန္းမအတြင္းရွိ Student Volunteer မ်ားအကူအညီျဖင့္ ထုတ္ယူခြင့္ျပဳပါသည္။

  • ၿပိဳင္ပြဲက်င္းပခ်ိန္ကို (၅)နာရီ သတ္မွတ္ ထားေသာ္လည္း မေမွ်ာ္လင့္ေသာ အေျခအေနမ်ား ျဖစ္ပါက ယင္းပြဲက်င္းပခ်ိန္ကို Judge မ်ားႏွင့္ ပြဲက်င္းပေရးအဖြဲ႔တို႔က ထပ္တိုးခြင့္ရွိပါသည္။ တိုးေပးသည့္ အခ်ိန္ကို ၿပိဳင္ပြဲ၀င္အသင္းမ်ားအားလံုး တူညီစြာ ခံစားခြင့္ရွိသည္။

  • ၿပိဳင္ပြဲက်င္းပေနစဥ္ တေလွ်ာက္လံုး ၿပိဳင္ပြဲ၀င္အသင္းမ်ား ရရွိသည့္ အဆင့္မ်ားကို အခ်ိန္ႏွင့္တေျပးညီ ထင္သာျမင္သာရွိေအာင္ Board မ်ားျဖင့္ ျပသေပးထားပါမည္။
